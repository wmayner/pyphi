``IIT3SystemIrreducibilityAnalysis`` now exposes the full set of phi-tied MIP candidates via the ``ties`` property (mirroring the IIT 4.0 ``SystemIrreducibilityAnalysis`` API). Before the ``PARTITION_LEX`` tiebreaker collapses the candidate set to a single winner, the partitions that tied at the maximum phi value are captured on the resulting SIA and round-trip through JSON via a ``_tie_peers`` payload. This lets fixtures and diagnostic consumers see the alternative valid MIPs instead of only the lex-chosen representative.
