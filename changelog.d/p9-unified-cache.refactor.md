The ``pyphi.cache`` module gained a uniform observability + control surface: ``pyphi.cache.info()`` returns per-cache statistics across the kernel ``_memoize`` caches and the module-level ``@cache(...)`` caches in ``pyphi.partition`` / ``pyphi.combinatorics`` / ``pyphi.distribution``; ``pyphi.cache.clear_all()`` and ``pyphi.cache.clear(name)`` clear them. The kernel cache now respects ``MAXIMUM_CACHE_MEMORY_PERCENTAGE`` (previously unbounded). ``Network.purview_cache`` instances stay anonymous (out of the registry); use ``network.purview_cache.info()`` directly for per-Network introspection. Caches are not thread-safe — PyPhi assumes process-isolated parallelism (Ray-based); future shared-memory parallelism will need locks.

The unused ``RedisCache`` class, ``REDIS_CACHE`` and ``REDIS_CONFIG`` config keys, the conftest Redis fixture, the ``CACHING.rst`` Redis documentation section, the benchmark Redis cache mode parameter, and the stale ``|MICECache|`` Sphinx alias have been removed. None had live call sites. Distributed caching is no longer supported; if you relied on the (never-wired) Redis scaffolding, you'll need to wait for a future feature that integrates with the new ``CachePolicy`` Protocol.
