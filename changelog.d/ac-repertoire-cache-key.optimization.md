The Actual Causation ``PRODUCT`` partitioned-repertoire scheme now computes its repertoires on the transition's underlying ``System`` rather than the ``TransitionSystem`` wrapper. The kernel repertoire cache keys on object identity, so passing the wrapper created a second cache entry for repertoires the unpartitioned path had already computed under the underlying ``System``; sharing the key lets coinciding ``(mechanism, purview)`` repertoires hit the cache instead of being recomputed. Accounts are unchanged.
