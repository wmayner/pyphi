``FactoredTPM`` and ``JointTPM`` now have a rich notebook/terminal display: a card showing the transition probabilities as a matrix grid (rows = current state in little-endian order), scrolling and showing an overflow indicator for large state spaces. Binary substrates show one column per unit (``P(next unit on | current state)``); non-binary (k-ary) substrates show one column per ``(unit, next-state)`` pair with the full conditional. Columns are labeled with node names when the TPM comes from a ``Substrate`` (e.g. ``P``/``Mc``/``Mn``), falling back to unit indices for a bare ``FactoredTPM``. Both types also gain ``to_xarray()`` for a labeled, dimension-sliceable view — a ``Dataset`` of per-unit conditionals for ``FactoredTPM``, a ``DataArray`` for ``JointTPM`` (requires the optional ``xarray`` dependency). The previous one-line ``repr`` is retained as the low-verbosity / embedded form.
