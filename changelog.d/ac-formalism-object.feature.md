Actual Causation now has a registered formalism object, ``AC2019Formalism``, mirroring the IIT formalism objects. It is selected by the new ``config.formalism.actual_causation.version`` setting (default ``"AC_2019"``) and validates the configured ``alpha_measure`` against its ``compatible_measures`` (``{"PMI", "WPMI"}``) at resolve time, matching IIT's behavior. The AC compute algorithms, the ``probability_distance`` / ``account_distance`` utilities, and the ``partitioned_repertoire_schemes`` / ``background_strategies`` / ``alpha_aggregations`` registries moved from ``pyphi.actual`` into the new ``pyphi.formalism.actual_causation`` package; import them from there (custom-scheme registration now uses ``pyphi.formalism.actual_causation.compute``). The public ``pyphi.actual.account`` / ``directed_account`` / ``sia`` API and the ``Transition`` methods are unchanged, and AC results are numerically unchanged. Because ``version`` now exists in both the ``iit`` and ``actual_causation`` config sub-namespaces, it is addressed by its qualified form (``config["iit.version"]`` / ``config["actual_causation.version"]``); nested ``pyphi_config.yml`` files route each sub-namespace's ``version`` (and other colliding fields) to the sub-namespace they are nested under on load.
