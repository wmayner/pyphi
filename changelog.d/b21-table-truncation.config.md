Collection tables in result displays (distinctions, relations, account links) now truncate to a configurable number of rows — ``config.infrastructure.repr_max_table_rows`` (default 50) — with a ``… N more`` indicator, so a result with thousands of distinctions no longer floods the terminal or bloats notebook output. The full data remains available via the object's iterables and ``to_pandas()``. In notebooks the (capped) table is also wrapped in a scrollable box.
