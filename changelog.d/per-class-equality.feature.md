Structural equality on IIT result objects is now precision-aware up to ``EQUALITY_TOLERANCE = 1e-13`` across all result-object classes: ``SystemIrreducibilityAnalysis`` (both 3.0 and 4.0), ``AcSystemIrreducibilityAnalysis``, ``AcRepertoireIrreducibilityAnalysis``, ``RepertoireIrreducibilityAnalysis``, ``StateSpecification``, ``Distinction``, ``Distinctions``, ``CauseEffectStructure``, and ``Relation``. The previous ``general_eq`` attribute-list dispatch is replaced with explicit per-class ``__eq__`` methods that directly call ``utils.eq`` for float scalars and ``numpy_aware_eq`` for arrays. Classes that also define ``__hash__`` (Distinction, RIA, AcRIA, StateSpecification, IIT 4.0 SIA, AcSIA, IIT 3.0 SIA) now use structural-only hashes (omitting tolerance-checked attributes) to satisfy Python's ``a == b → hash(a) == hash(b)`` contract under tolerance-aware equality. ``Orderable.unorderable_unless_eq`` ClassVar is replaced with an ``is_orderable_with(other)`` method; subclasses that need cross-instance guards (``AcSystemIrreducibilityAnalysis``, ``AcRepertoireIrreducibilityAnalysis``, ``CausalLink``) provide explicit overrides.
