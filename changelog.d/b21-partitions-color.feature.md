Partitions/cuts now have a rich standalone display: a card with the type's shorthand headline plus a labeled "severed connections" grid (rows = from-node, cols = to-node, ``✕`` marks a cut edge), built on the universal ``removed_edges()``. When embedded in another result (a SIA's MIP row, an RIA, an account link), a partition still shows its concise one-line shorthand. Additionally, cause and effect are now color-coded in notebook HTML output (cause ``#D55C00``, effect ``#009E73`` — colorblind-safe) across SIAs, distinctions, state specifications, RIA/MICE, actual-causation results, and directed partitions; terminal/ASCII output is unchanged.
