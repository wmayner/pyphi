Tied-set metadata now round-trips through ``to_json`` / ``from_json`` for ``StateSpecification``, ``RepertoireIrreducibilityAnalysis`` (both ``_partition_ties`` and ``_state_ties``), and ``SystemIrreducibilityAnalysis``. Peers are emitted as the canonical winner's ``_tie_peers`` (or ``_partition_tie_peers`` / ``_state_tie_peers`` for RIA) — a list of shallow peer dicts that bypass the jsonify cache (whose ``hash``-keyed deduplication would otherwise collapse partition-tied peers into a single object). After deserialization, all members of a tied set share the same tuple. ``warn_about_tie_serialization`` and its emitted ``PyPhiWarning`` are removed; consumers serializing tied analyses no longer see a warning.
