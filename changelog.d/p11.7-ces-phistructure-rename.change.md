**Paper-aligned model-class rename for the cause-effect structure
hierarchy** (Albantakis et al. 2023, p11). The IIT 4.0 paper distinguishes
*cause-effect structure* (distinctions + relations of any candidate system)
from *Φ-structure* (the cause-effect structure of a *complex* — a maximally
irreducible substrate). PyPhi's previous class names conflated these:

- The old ``CauseEffectStructure`` (a sequence of concepts/distinctions, no
  relations) is now :class:`pyphi.models.Distinctions`. The file moved
  from ``pyphi/models/ces.py`` to ``pyphi/models/distinctions.py``.
- The old ``PhiStructure`` (a dataclass holding distinctions + relations +
  SIA — paper-faithful for the cause-effect structure of any candidate
  system, not just complexes) is now
  :class:`pyphi.models.CauseEffectStructure`. The file moved from
  ``pyphi/models/phi_structure.py`` to ``pyphi/models/ces.py``.
- The term *Φ-structure* now lives only as a docstring/conceptual term used
  in SIA contexts to communicate the additional complex invariant ("the
  cause-effect structure of a complex"); it is no longer a separate
  runtime type. The :attr:`SystemIrreducibilityAnalysis.phi_structure`
  attribute name is paper-faithful for the SIA-winner case and is
  preserved.

Test fixture JSON keys (``"CauseEffectStructure"``, ``"PhiStructure"``)
are migrated to the new names. No back-compatibility shims — pre-release
rename.
