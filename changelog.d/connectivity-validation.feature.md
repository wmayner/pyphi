Substrate construction now validates the connectivity matrix against the TPM. A connectivity matrix that *omits* an edge the TPM actually implies (an under-specified matrix) previously caused that dependency to be silently marginalized out, under-counting ``phi`` with no error. Such matrices are now rejected with a ``ValueError`` naming the missing edge(s). The check is directional and k-ary-aware (an edge ``a -> b`` exists iff node ``b``'s conditional distribution depends on node ``a``'s state) and includes self-loops; over-specification — declaring an edge the TPM does not use — remains legal, since it only widens the search. The new ``FactoredTPM.infer_cm()`` / ``infer_edge()`` methods infer the implied connectivity directly from the factors. The validation is on by default; set ``config.infrastructure.validate_connectivity = False`` to allow a deliberately permissive connectivity matrix — for example an input unit whose TPM column copies its own previous state but whose connectivity matrix intentionally omits the self-loop so that its effect repertoire is left unconstrained.
