Subsystem internals rewritten as a layered architecture in `pyphi.core` and `pyphi.formalism`. The new `CandidateSystem` value type (`(CausalModel, state, node_indices, cut)`) replaces the legacy `Subsystem` class. Repertoire computations are stateless module-level functions in `pyphi.core.repertoire_algebra` (the kernel), with module-level memoization keyed on `id(cs)` and weakref-based cache eviction. Formalism queries (`find_mip`, `sia`, `cause_mip`, `effect_mip`, `phi`, `find_mice`, `mic`, `mie`, `phi_max`, `concept`, `distinction`, `all_distinctions`, `evaluate_partition`) live as free functions in `pyphi.formalism` — call as `formalism.find_mip(cs, ...)` rather than `cs.find_mip(...)`. CandidateSystem retains sugar proxies for kernel operations only (cause/effect/forward repertoires, info, intrinsic_information). The legacy `Subsystem` class is preserved during this transitional release for test compatibility but is no longer used by the formalism or compute layers.
