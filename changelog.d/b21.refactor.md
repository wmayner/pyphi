Unified all result-object display behind a single declarative ``_describe()`` model in the new ``pyphi.display`` package, rendered by pluggable backends. Every user-facing result type (SIAs, ``CauseEffectStructure``/``PhiFold``, distinctions, relations, RIA/MICE, partitions, ``Complex``, ``System``/``Substrate``/``Node``, actual-causation results, state specifications) now has a consistent, redesigned ``repr``/``str`` and a styled ``_repr_html_`` for notebooks: a boxed-card terminal layout with grouped sections, readable rounded numbers, and an HTML-native card (header, key/value grids, real tables for collections). Replaces the per-class ``fmt.py`` composers; ``repr_verbosity`` is now read in one place.
