# IIT 3.0 Cross-Subsystem Complex Tie Resolution

## Problem

When two candidate subsystems tie at maximum |big_phi| during major-complex
selection, PyPhi's IIT 3.0 path picks a winner using the within-subsystem
MIP's `partition.lex_key()` as a secondary key on `OrderableByPhi`. This
is incidental: a partition object's lex key (a property of cuts inside one
subsystem) is being used to choose between two different subsystems. The
choice is reproducible but not principled.

The worked example: substrate `standard` at `current_state=(0, 0, 1)` has
two SIAs that genuinely tie at |big_phi| = 1.0 — the subsystem over nodes
`(1, 2)` (B=AND, C=XOR) and the subsystem over nodes `(0, 2)` (A=OR,
C=XOR). These are physically distinct subsystems with different gate
compositions, not symmetry equivalents. PyPhi 1.x picked `(1, 2)` by
iteration order; the post-`82b778ca` `partition.lex_key()` cascade picks
`(0, 2)`. The Albantakis et al. 2019 paper explicitly notes (p. 17) that
symmetric ties of this form are "undetermined" — neither winner is
paper-canonical.

The downstream consequence is observable: for the transition
`(1, 0, 0) → (0, 0, 1) → (1, 1, 0)`, `true_events(substrate, *states)`
returns 2 events under `(1, 2)` and 0 events under `(0, 2)`.
`test/test_actual.py::TestActualCausationIIT30::test_true_events` is
currently `@pytest.mark.skip` pending an explicit policy.

PyPhi must pick deterministically. The choice is a policy decision; the
remaining work is to (a) commit to a policy, (b) wire it through a
principled key, (c) unskip the test, (d) document the convention and the
related tie-resolution knobs that do or don't apply on the IIT 3.0 path.

## Goals

1. Replace the incidental `partition.lex_key()` cross-subsystem
   tie-break with a principled, documented Determinism-level key.
2. Unskip `test_true_events` with updated expectations matching the
   chosen policy.
3. Document the IIT 3.0 tie-resolution knob landscape in
   `presets.iit3` so future readers see which paths are
   config-driven, which are hard-coded, and which are no-ops on
   3.0.

## Non-goals (deferred to post-2.0)

- IIT 3.0 system-MIP-within-subsystem: stays on
  `IIT3SystemIrreducibilityAnalysis.order_by = (phi,
  partition.lex_key())`. Tononi 2008 `Cut.normalized_phi`
  (`Φ / min(|M¹|, |M²|)`) integration is post-2.0 work; it requires
  either a new `distinction_phi_normalization` registry entry or a
  system-level normalization computation that isn't currently
  surfaced.
- Routing `sia_tie_resolution` config through IIT 3.0's
  `_sia_map_reduce`: post-2.0 architectural unification.
- Changing `mip_tie_resolution` default for IIT 3.0 (the current
  `["NORMALIZED_PHI", "NEGATIVE_PHI"]` key uses IIT 4.0
  `NUM_CONNECTIONS_CUT` normalization, well-defined but not
  paper-canonical for 3.0; deterministic).

## Tie-resolution paths on the IIT 3.0 path

Four distinct paths with three knobs and one hard-coded layer:

| Path | Where | Mechanism | Config knob | Status |
|---|---|---|---|---|
| Mechanism MIP | `formalism/queries.py:find_partition_mip` | `resolve_ties.partitions` | `mip_tie_resolution` | Shared with 4.0; no change |
| Purview at MICE | `formalism/queries.py:289` | `resolve_ties.purviews` | `purview_tie_resolution` | Already `["PHI", "PURVIEW_SIZE"]` in preset |
| System MIP within subsystem | `formalism/iit3/__init__.py:_sia_map_reduce` | `MapReduce(reduce_func=min)` over `IIT3SIA.order_by` | none (hard-coded) | No change for 2.0 |
| Cross-subsystem complex | `substrate.complexes` → `_greedy_condensation` | `sorted()` over `IIT3SIA.__lt__` | none | **This spec changes this layer** |

The cross-subsystem layer is the only one the worked example actually
touches; the other three are listed so the documentation can enumerate
them.

## Policy

When two subsystems tie at maximum |big_phi| during major-complex
selection on the IIT 3.0 path, PyPhi selects the subsystem whose node
indices form the lex-smallest tuple. For the worked example, `(0, 2)`
wins over `(1, 2)` because `(0, 2) < (1, 2)` in tuple lex order.

**Rationale.** The Albantakis et al. 2019 paper does not prescribe a
system-level tie-break — any pyphi convention is a Determinism-level
canonicalization, not a paper claim. Among defensible conventions:

- **Lex-smallest node_indices** (proposed) — matches the existing AC
  cascade's Determinism step in `resolve_ac_causal_link_tie` (which
  picks the lex-smallest purview as the canonical representative
  among indeterminate actual causes). Picks `(0, 2)` for the worked
  example, which is also the answer the current incidental
  `partition.lex_key()` route produces. Fewest goldens to regenerate.
- Lex-largest node_indices — restores PyPhi 1.x's iteration-order
  answer; less consistent with the AC cascade pattern.

Lex-smallest is chosen for architectural consistency with the existing
cascade Determinism pattern and for minimal regression impact.

## Architecture

Single new cascade resolver `resolve_iit3_complex_tie` in
`pyphi/resolve_ties.py`, wired into a new `_iit3_exclusion_cascade` in
`pyphi/substrate.py` that mirrors the existing
`_substrate_exclusion_cascade` (tier walk, overlap-clique grouping)
without the Composition |big_phi| escalation. `complexes()` dispatches
on `config.formalism.iit.version`.

### Cascade resolver

```python
def resolve_iit3_complex_tie[V: _IIT3SiaLike](
    sias: "Iterable[V]",
    *,
    context: ResolutionContext,
    on_unresolved: OnUnresolved = "defer",
) -> CascadeOutcome[V]:
    """Resolve a cross-subsystem complex tie on the IIT 3.0 path.

    The Albantakis et al. 2019 paper does not prescribe a system-level
    tie-break for IIT 3.0; this is a pyphi Determinism-level
    canonicalization. The cascade picks the candidate whose node
    indices form the lex-smallest tuple.
    """
    return cascade(
        sias,
        levels=[
            CascadeLevel(
                postulate="Determinism",
                op="argmin",
                key=lambda s: tuple(sorted(s.node_indices)),
            ),
        ],
        context=context,
        on_unresolved=on_unresolved,
    )
```

The `_IIT3SiaLike` protocol declares `node_indices: Sequence[int]` and
nothing else.

### Substrate-level wiring

```python
def _resolve_clique_iit3(clique: list[Any]) -> Any | None:
    """Resolve an overlap clique on the IIT 3.0 path via the
    Determinism-level cascade.
    """
    ctx = resolve_ties.ResolutionContext(max_escalation_level="Determinism")
    outcome = resolve_ties.resolve_iit3_complex_tie(clique, context=ctx)
    return outcome.resolved if outcome.outcome == "RESOLVED" else None


def _iit3_exclusion_cascade(
    sorted_sias: list[Any],
    substrate: Substrate,
    state: tuple[int, ...],
) -> list[Any]:
    """Walk SIAs in descending |big_phi| tiers, applying the IIT 3.0
    Determinism-level cascade within each overlap clique.
    """
    result: list[Any] = []
    covered: set[int] = set()
    for tier in _phi_groups(sorted_sias):
        survivors = [
            sia for sia in tier
            if not (set(_sia_node_indices(sia) or ()) & covered)
        ]
        if not survivors:
            continue
        for clique in _find_overlap_cliques(survivors):
            if len(clique) == 1:
                _accept(clique[0], result, covered)
                continue
            winner = _resolve_clique_iit3(clique)
            if winner is not None:
                _accept(winner, result, covered)
    return result
```

`complexes()` dispatch becomes:

```python
if _config_iit_version() == "IIT_3_0":
    return _iit3_exclusion_cascade(sorted_sias, substrate, state)
return _substrate_exclusion_cascade(sorted_sias, substrate, state)
```

The existing `_greedy_condensation` is no longer reached and is
removed.

### Optional registry entry

For surface consistency with the existing
`phi_object_tie_resolution_strategies` registry, add a
`NODE_INDICES_LEX` entry:

```python
@phi_object_tie_resolution_strategies.register("NODE_INDICES_LEX")
def _(sia: Any) -> tuple[int, ...]:
    return tuple(sorted(sia.node_indices))
```

This is not consulted by the new cascade (the cascade uses the key
directly) but registers the name for future config-driven extension
and for `pyphi.config.formalism.iit.mip_tie_resolution = ["...",
"NODE_INDICES_LEX"]`-style overrides.

## Risk and rollback

- **Risk**: a golden depends on the cross-subsystem tie-break in a
  way the audit misses. *Mitigation*: regenerate after running the
  full suite; commit the regeneration alongside the policy change
  so the diff makes the dependency explicit.
- **Risk**: the lex-smallest policy is later judged wrong (e.g.,
  user feedback prefers PyPhi 1.x's lex-largest). *Mitigation*: the
  cascade key is one line; switching to argmax is a one-character
  diff plus golden re-regeneration. The decision is reversible.
- **Risk**: `_greedy_condensation` removal breaks an external
  caller. *Mitigation*: it's a private module-level function; grep
  confirms no external imports.
